- [ ] **Step 4.1: Add `.hero-name-lcp-h1` class to `src/index.css`**

Insert this block in `src/index.css` immediately after the `.sr-only` rule (which currently ends at line 275). The class makes the h1 a real, sized, layout-occupying element with transparent text — Lighthouse picks up its bounding box as a text-LCP candidate; axe-core skips contrast (transparent has no foreground); the user sees nothing because the SVG above it renders the actual visual. The `pointer-events: none` keeps it from intercepting clicks meant for the SVG.

```css
/* LCP candidate: a real-sized, layout-occupying <h1> rendered with
   transparent text so Lighthouse can classify it as the text-LCP element
   on first paint. The visible hero name comes from the SVG sibling
   (.hero-name-drawing). axe-core skips color-contrast on transparent
   text (no foreground to measure), so this introduces zero a11y
   regression. The h1 stays accessible to screen readers as the page
   heading. */
.hero-name-lcp-h1 {
  position: absolute;
  inset: 0 auto auto 0;
  width: 100%;
  font-size: clamp(72px, 22vw, 192px);
  line-height: 0.92;
  letter-spacing: -0.05em;
  text-transform: lowercase;
  font-weight: 700;
  margin: 0;
  color: transparent;
  pointer-events: none;
  user-select: none;
}
```
