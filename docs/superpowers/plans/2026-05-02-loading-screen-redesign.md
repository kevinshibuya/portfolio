# Loading Screen Redesign Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` (recommended) or `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking. **After each step's command lands successfully, edit the corresponding `- [ ]` to `- [x]` in this plan file before proceeding to the next step.** When a task completes, also tick the matching `- [ ]` in the spec's `## TODO` list (`docs/superpowers/specs/2026-05-02-loading-screen-redesign-design.md`).

**Goal:** Replace the fixed-panel mirror-of-hero loader with an in-place SVG-drawn name paced by load progress, a periwinkle cursor dot that glides along the active baseline, and a post-draw choreography (ink-fill → period bloom → float-to-nav → identity merge with the nav availability dot).

**Architecture:** Hero owns its own pre-draw state. `<HeroNameDrawing>` renders 13 glyphs of `kevin shibuya.` as `<motion.path>` strokes; their `pathLength` is interpolated from a single `progress` value (0..1) sourced from a new `useLoaderProgress` hook. A body-portal'd `<LoadingCursor>` follows the active baseline, blooms on the period at `progress === 1`, then flies to the nav availability dot's runtime-measured position. Hero's existing `RevealOnView gate={gate}` cascade now gates on a new `handoffDone` promise (which `loaderDone` is aliased to, so existing consumers don't change). Glyph SVG path data is extracted once at build time from Plus Jakarta Sans via opentype.js and committed as a static TS constant.

**Tech Stack:** React 19, TypeScript strict, Vite 6 + SWC, framer-motion v12 (motion.path + pathLength), opentype.js (build-time only), TailwindCSS v4, vitest, Playwright.

**Spec:** `docs/superpowers/specs/2026-05-02-loading-screen-redesign-design.md` (commit `dad04e1`).

This task adds a one-shot Node script that reads `public/fonts/PlusJakartaSans-VariableFont_wght.ttf`, walks the string `kevin shibuya.`, and writes per-glyph SVG path data to `src/data/glyphPaths.ts`. The string is fixed (the name is the same in EN and PT) so the extraction never needs to re-run unless the font itself changes.

opentype.js renders variable fonts at their default instance (weight 400). The hero displays at weight 700; we accept this visual delta on the *outline* shape during the draw because the strokes will be filled to weight-700 ink color at the end of the draw, where weight matters more (filled glyphs vs outlined). If verification (Task 10) shows the outline silhouette feels too light, the escape hatch is to bake a static weight-700 TTF via fonttools and re-run extraction; that's documented as a follow-up, not blocking.

```js
// scripts/extract-glyph-paths.mjs
import { readFileSync, writeFileSync } from 'node:fs'
import { resolve, dirname } from 'node:path'
import { fileURLToPath } from 'node:url'
import opentype from 'opentype.js'

const __dirname = dirname(fileURLToPath(import.meta.url))
const ROOT = resolve(__dirname, '..')

const FONT_PATH = resolve(ROOT, 'public/fonts/PlusJakartaSans-VariableFont_wght.ttf')
const OUT_PATH = resolve(ROOT, 'src/data/glyphPaths.ts')

// Two words rendered separately (no inter-word space — they live on different
// baselines in the hero). Period is its own glyph at the end of `shibuya.`.
const KEVIN = 'kevin'
const SHIBUYA = 'shibuya.'

// Render glyphs at a normalized 1000-unit em. The SVG's viewBox + CSS width
// scale to whatever pixel size the hero <h1> would be at the current viewport.
const FONT_SIZE = 1000

const fontBuffer = readFileSync(FONT_PATH)
const font = opentype.parse(fontBuffer.buffer.slice(
  fontBuffer.byteOffset,
  fontBuffer.byteOffset + fontBuffer.byteLength
))

function extract(word) {
  const glyphs = []
  let cursorX = 0
  for (const ch of word) {
    const glyph = font.charToGlyph(ch)
    const path = glyph.getPath(cursorX, 0, FONT_SIZE)
    const advance = (glyph.advanceWidth / font.unitsPerEm) * FONT_SIZE
    glyphs.push({
      char: ch,
      d: path.toPathData(2),
      advance,
      x: cursorX,
    })
    cursorX += advance
  }
  return { glyphs, totalAdvance: cursorX }
}

const kevin = extract(KEVIN)
const shibuya = extract(SHIBUYA)

const ts = `// AUTO-GENERATED by scripts/extract-glyph-paths.mjs. Do not hand-edit.
// To regenerate: \`node scripts/extract-glyph-paths.mjs\`

export interface GlyphPath {
  /** The character this glyph represents (for debugging / a11y aside). */
  char: string
  /** SVG \`d\` attribute. Coordinates are in the 1000-unit em space and
   *  pre-positioned along the word's x-axis (cursorX accumulated). */
  d: string
  /** x-advance in em-units; used by callers if they need to position
   *  decorations relative to a single glyph. */
  advance: number
  /** x-origin of this glyph within its word (em-units). */
  x: number
}

export interface NameWord {
  glyphs: GlyphPath[]
  /** Total x-advance of the word in em-units. */
  totalAdvance: number
}

/** Em-size all coordinates are normalized to. */
export const NAME_FONT_EM = ${FONT_SIZE}

export const NAME_KEVIN: NameWord = ${JSON.stringify(kevin, null, 2)}

export const NAME_SHIBUYA: NameWord = ${JSON.stringify(shibuya, null, 2)}
`

writeFileSync(OUT_PATH, ts)
console.log(`Wrote ${OUT_PATH}`)
console.log(`  kevin: ${kevin.glyphs.length} glyphs, totalAdvance=${kevin.totalAdvance.toFixed(1)}`)
console.log(`  shibuya.: ${shibuya.glyphs.length} glyphs, totalAdvance=${shibuya.totalAdvance.toFixed(1)}`)
```

## Task 2: `useLoaderProgress` hook

**Files:**
- Create: `src/hooks/useLoaderProgress.ts`
- Create: `tests/unit/useLoaderProgress.test.tsx`

Single source of truth for the `progress` value (0..1) and the `drawDone` / `handoffDone` promises. Synthetic ramp toward 0.92 over `LOADER_MIN_DURATION_MS`; snaps to 1.0 once `window 'load'` fires AND `document.fonts.ready` resolves; reduced-motion path snaps to 1 after `LOADER_REDUCED_MOTION_MAX_MS` and resolves both promises immediately.

Critically: the hook does NOT own the `handoffDone` resolution. `handoffDone` resolves later, when the cursor's flight to the nav completes — `LoadingCursor` (Task 5) calls back into the hook to resolve it. The hook returns `resolveHandoff` for that callback.

```ts
// src/hooks/useLoaderProgress.ts
import { useEffect, useMemo, useRef, useState } from 'react'
import { animate } from 'framer-motion'
import {
  LOADER_MIN_DURATION_MS,
  LOADER_REDUCED_MOTION_MAX_MS,
} from '../utils/motion-flags'

export interface LoaderProgress {
  /** 0..1, monotonically non-decreasing. */
  progress: number
  /** Resolves when progress reaches 1 (the draw is visually complete). */
  drawDone: Promise<void>
  /** Resolves when the cursor's flight to the nav completes. The hook does
   *  not resolve this itself — LoadingCursor calls resolveHandoff() after
   *  its flight animation finishes. Under reduced motion the hook resolves
   *  it immediately alongside drawDone. */
  handoffDone: Promise<void>
  resolveHandoff: () => void
}

const POWER3_OUT: [number, number, number, number] = [0.165, 0.84, 0.44, 1]

export function useLoaderProgress(prefersReducedMotion: boolean): LoaderProgress {
  const [progress, setProgress] = useState(0)

  // Stable promise instances per hook lifecycle.
  const promises = useMemo(() => {
    let resolveDraw!: () => void
    let resolveHandoff!: () => void
    const drawDone = new Promise<void>((r) => { resolveDraw = r })
    const handoffDone = new Promise<void>((r) => { resolveHandoff = r })
    return { drawDone, handoffDone, resolveDraw, resolveHandoff }
  }, [])

  const resolveHandoffRef = useRef(promises.resolveHandoff)

  useEffect(() => {
    const start = performance.now()
    const minDelay = prefersReducedMotion
      ? LOADER_REDUCED_MOTION_MAX_MS
      : LOADER_MIN_DURATION_MS

    let cancelled = false
    let snapControls: ReturnType<typeof animate> | null = null

    // Synthetic ramp: progress ticks up to a 0.92 ceiling on a raf cadence.
    // Capped at 0.92 so the visual still shows "not quite done" until the
    // real load signals fire — they then drive the final snap to 1.0.
    let rafId = 0
    const tick = () => {
      if (cancelled) return
      const elapsed = performance.now() - start
      const synthetic = Math.min(0.92, elapsed / minDelay)
      setProgress((p) => Math.max(p, synthetic))
      if (!cancelled) rafId = requestAnimationFrame(tick)
    }
    rafId = requestAnimationFrame(tick)

    const finish = (): void => {
      if (cancelled) return
      // Stop the synthetic ramp and tween from current value to 1.0 over 240ms.
      cancelled = true
      cancelAnimationFrame(rafId)
      const fromValue = Math.min(0.92, (performance.now() - start) / minDelay)
      // Under reduced motion we skip the tween — snap immediately.
      if (prefersReducedMotion) {
        setProgress(1)
        promises.resolveDraw()
        promises.resolveHandoff()
        return
      }
      snapControls = animate(fromValue, 1, {
        duration: 0.24,
        ease: POWER3_OUT,
        onUpdate: (v) => setProgress(v),
        onComplete: () => promises.resolveDraw(),
      })
    }

    const onLoad = (): void => {
      void document.fonts.ready.then(() => {
        if (document.readyState === 'complete') finish()
        else window.addEventListener('load', () => finish(), { once: true })
      })
    }

    if (document.readyState === 'complete') {
      onLoad()
    } else {
      window.addEventListener('load', onLoad, { once: true })
    }

    // Reduced-motion floor: even if assets are slow, snap by minDelay.
    let floorTimer: number | undefined
    if (prefersReducedMotion) {
      floorTimer = window.setTimeout(finish, minDelay)
    }

    return () => {
      cancelled = true
      cancelAnimationFrame(rafId)
      snapControls?.stop()
      if (floorTimer !== undefined) window.clearTimeout(floorTimer)
      window.removeEventListener('load', onLoad)
    }
  }, [prefersReducedMotion, promises])

  return {
    progress,
    drawDone: promises.drawDone,
    handoffDone: promises.handoffDone,
    resolveHandoff: () => resolveHandoffRef.current(),
  }
}
```

## Task 3: Extend `MotionContext` with progress and handoff

**Files:**
- Modify: `src/context/MotionContext.tsx`

`MotionContext` is upgraded to host the loader progress signal. The existing `loaderDone: Promise<void>` is preserved for backward compatibility but now points to the new `handoffDone` (which resolves at the same conceptual moment — page is interactive, hero can cascade in).

The hook must mount inside the React tree, so we move it into `MotionProvider` and replace the module-scoped `_loaderDone` Promise. **This means HMR no longer preserves the loader promise across remounts** — acceptable because every HMR cycle restarts the loader anyway, and there are no consumers awaiting the promise from outside the provider's lifetime.

- [ ] **Step 1: Rewrite `MotionContext.tsx`**

```tsx
// src/context/MotionContext.tsx
import { createContext, useContext, useEffect, useMemo, useState } from 'react'
import { useReducedMotion } from 'framer-motion'
import { ENABLE_R3F_ACCENT, MOBILE_BREAKPOINT_PX } from '../utils/motion-flags'
import { useLoaderProgress } from '../hooks/useLoaderProgress'

interface MotionContextValue {
  /** 0..1 loader progress; drives the HeroNameDrawing stroke pathLengths and
   *  the LoadingCursor's X position during the draw phase. */
  progress: number
  /** Resolves when progress reaches 1 (visual draw complete, before
   *  ink-fill / period bloom / float-to-nav). HeroNameDrawing awaits this
   *  to start its ink-fill transition. */
  drawDone: Promise<void>
  /** Resolves when the cursor's flight to the nav completes. Hero awaits
   *  this to swap from <HeroNameDrawing> to <h1> and to fire the
   *  RevealOnView cascade for role/desc/CTA/accent. */
  handoffDone: Promise<void>
  /** Backward-compat alias for handoffDone. Existing consumers that read
   *  `loaderDone` continue to work without code changes. */
  loaderDone: Promise<void>
  /** Called by LoadingCursor when its flight animation completes. */
  resolveHandoff: () => void
  prefersReducedMotion: boolean
  r3fAccentEnabled: boolean
}

const Ctx = createContext<MotionContextValue | null>(null)

export function MotionProvider({ children }: { children: React.ReactNode }) {
  const reduced = useReducedMotion() ?? false

  const { progress, drawDone, handoffDone, resolveHandoff } =
    useLoaderProgress(reduced)

  const [r3fEnabled, setR3fEnabled] = useState(false)
  useEffect(() => {
    const url = new URL(window.location.href)
    const forceOff = url.searchParams.get('disableR3f') === '1'
    setR3fEnabled(ENABLE_R3F_ACCENT && !forceOff && window.innerWidth >= MOBILE_BREAKPOINT_PX)
  }, [])

  const value = useMemo<MotionContextValue>(
    () => ({
      progress,
      drawDone,
      handoffDone,
      loaderDone: handoffDone,
      resolveHandoff,
      prefersReducedMotion: reduced,
      r3fAccentEnabled: r3fEnabled,
    }),
    [progress, drawDone, handoffDone, resolveHandoff, reduced, r3fEnabled]
  )

  return <Ctx.Provider value={value}>{children}</Ctx.Provider>
}

export function useMotion(): MotionContextValue {
  const v = useContext(Ctx)
  if (!v) throw new Error('useMotion must be used within MotionProvider')
  return v
}
```

## Task 4: `HeroNameDrawing` component

**Files:**
- Create: `src/components/ui/HeroNameDrawing.tsx`
- Create: `tests/unit/HeroNameDrawing.test.tsx`

SVG component that renders all 13 glyphs (5 + 8 with period included in shibuya) as `<motion.path>` elements. Per-glyph `pathLength` is computed from a global `progress` value and a stagger window: `glyph N (of 13)` gets window `[N × 0.05, N × 0.05 + 0.4]` clamped to `[0, 1]`. The 13th glyph (period, N=12) finishes at `0.05 × 12 + 0.4 = 1.0` exactly.

After `progress === 1` and `drawDone` resolves, the component runs a 400ms cross-fade transitioning each glyph's `stroke` and `fill`:
- `kevin` glyphs → `stroke: transparent, fill: var(--ink)`
- `shibuya.` glyphs → `stroke: var(--blue-300), fill: var(--mist)`

Pre-handoff, the component also exposes its computed bounding boxes (kevin baseline, shibuya baseline, period coordinates) via a callback ref so `LoadingCursor` can position itself relative to the SVG. We use `useImperativeHandle` to expose a `getCursorAnchors()` method.

A screen-reader-only `<h1 className="sr-only">kevin shibuya.</h1>` is rendered alongside the SVG so the name is announced from frame 1.

The SVG sizing matches `.hero-name`'s `font-size: clamp(64px, 11vw, 192px)`. The viewBox is determined by the kevin/shibuya em-units; CSS sets the height via `font-size`-equivalent measurement and width auto-sizes from the viewBox aspect ratio. Practically: render two stacked SVGs (one per word) since they have different widths and different baselines; this keeps each word's viewBox independent and correct.

```tsx
// src/components/ui/HeroNameDrawing.tsx
import { forwardRef, useImperativeHandle, useMemo, useRef } from 'react'
import { motion } from 'framer-motion'
import { useMotion } from '../../context/MotionContext'
import { NAME_KEVIN, NAME_SHIBUYA, NAME_FONT_EM } from '../../data/glyphPaths'

/** Coordinates LoadingCursor needs to position itself. All values in
 *  viewport pixel space (post-getBoundingClientRect). Recomputed by the
 *  caller on resize. */
export interface CursorAnchors {
  kevinStartX: number
  kevinEndX: number
  kevinBaselineY: number
  shibuyaStartX: number
  shibuyaEndX: number
  shibuyaBaselineY: number
  periodX: number
  periodY: number
}

export interface HeroNameDrawingHandle {
  /** Returns current viewport-space anchors for the cursor, or null if the
   *  SVGs have not yet been mounted/measured. */
  getCursorAnchors: () => CursorAnchors | null
}

const STAGGER_PER_GLYPH = 0.05
const DRAW_WINDOW = 0.40
const TOTAL_GLYPHS = NAME_KEVIN.glyphs.length + NAME_SHIBUYA.glyphs.length // 13

function glyphPathLength(progress: number, globalIndex: number): number {
  const start = globalIndex * STAGGER_PER_GLYPH
  const end = start + DRAW_WINDOW
  if (progress <= start) return 0
  if (progress >= end) return 1
  return (progress - start) / (end - start)
}

export const HeroNameDrawing = forwardRef<HeroNameDrawingHandle>(function HeroNameDrawing(_, ref) {
  const { progress, drawDone } = useMotion()
  const kevinSvgRef = useRef<SVGSVGElement>(null)
  const shibuyaSvgRef = useRef<SVGSVGElement>(null)

  // After drawDone, transition stroke→fill. We track this via a derived
  // boolean: if progress >= 1 then we're in the ink-fill phase.
  const inkFilled = progress >= 1

  // Promise side-effect for downstream (Hero crossfade) — drawDone is the
  // signal; nothing for HeroNameDrawing itself to do beyond reading progress.
  void drawDone

  useImperativeHandle(ref, () => ({
    getCursorAnchors: () => {
      const k = kevinSvgRef.current
      const s = shibuyaSvgRef.current
      if (!k || !s) return null
      const kRect = k.getBoundingClientRect()
      const sRect = s.getBoundingClientRect()
      // Period is the last glyph in shibuya. Compute its viewport X by
      // ratio of (period.x + period.advance/2) within the shibuya SVG
      // viewBox, mapped through the SVG's screen width.
      const period = NAME_SHIBUYA.glyphs[NAME_SHIBUYA.glyphs.length - 1]
      const periodRatioX = (period.x + period.advance * 0.4) / NAME_SHIBUYA.totalAdvance
      return {
```

```tsx
// src/components/ui/LoadingCursor.tsx
import { useEffect, useState } from 'react'
import { createPortal } from 'react-dom'
import { animate } from 'framer-motion'
import { useMotion } from '../../context/MotionContext'
import type { CursorAnchors } from './HeroNameDrawing'

interface LoadingCursorProps {
  /** Called per render to get current viewport-space anchors. Returning
   *  null skips rendering until the SVGs have been measured. */
  getAnchors: () => CursorAnchors | null
}

const KEVIN_PHASE_END = 0.45  // approx — derived from stagger; see spec §5
const PEN_LIFT_HALF = 0.03    // ±0.03 around KEVIN_PHASE_END

interface DotState {
  x: number
  y: number
  opacity: number
}

function computeDuringDraw(
  progress: number,
  a: CursorAnchors
): DotState {
  if (progress < KEVIN_PHASE_END - PEN_LIFT_HALF) {
    const t = progress / (KEVIN_PHASE_END - PEN_LIFT_HALF)
    return {
      x: a.kevinStartX + t * (a.kevinEndX - a.kevinStartX),
      y: a.kevinBaselineY + 14,
      opacity: 1,
    }
  }
  if (progress < KEVIN_PHASE_END + PEN_LIFT_HALF) {
    const opacity =
      progress < KEVIN_PHASE_END
        ? 1 - (progress - (KEVIN_PHASE_END - PEN_LIFT_HALF)) / PEN_LIFT_HALF
        : (progress - KEVIN_PHASE_END) / PEN_LIFT_HALF
    return { x: 0, y: 0, opacity: Math.max(0, opacity) }
  }
  if (progress < 1) {
    const t =
      (progress - (KEVIN_PHASE_END + PEN_LIFT_HALF)) /
      (1 - (KEVIN_PHASE_END + PEN_LIFT_HALF))
    return {
      x: a.shibuyaStartX + t * (a.shibuyaEndX - a.shibuyaStartX),
      y: a.shibuyaBaselineY + 14,
      opacity: 1,
    }
  }
  return { x: a.periodX, y: a.periodY, opacity: 1 }
}

export function LoadingCursor({ getAnchors }: LoadingCursorProps) {
  const { progress, drawDone, prefersReducedMotion, resolveHandoff } = useMotion()
  const [haloScale, setHaloScale] = useState(0.6)
  const [haloOpacity, setHaloOpacity] = useState(0)
  const [dotScale, setDotScale] = useState(1)
  const [flightPos, setFlightPos] = useState<{ x: number; y: number } | null>(null)
  const [postFlightOpacity, setPostFlightOpacity] = useState(1)
  const [unmounted, setUnmounted] = useState(false)

  useEffect(() => {
    if (prefersReducedMotion) return
    let cancelled = false

    void drawDone.then(() => {
      if (cancelled) return
      // Period bloom: dot scales 1 → 1.15 → 1; halo expands 0.6 → 3 with
      // opacity 0 → 0.4 → 0 over 500ms.
      animate(1, 1.15, {
        duration: 0.25,
        ease: 'easeOut',
        onUpdate: (v) => setDotScale(v),
        onComplete: () => {
          animate(1.15, 1, {
            duration: 0.25,
            ease: 'easeIn',
            onUpdate: (v) => setDotScale(v),
          })
        },
      })
      animate(0.6, 3, {
        duration: 0.5,
        ease: 'easeOut',
        onUpdate: (v) => setHaloScale(v),
      })
      animate(0, 0.4, {
        duration: 0.18,
        ease: 'easeOut',
        onUpdate: (v) => setHaloOpacity(v),
        onComplete: () => {
          animate(0.4, 0, {
            duration: 0.32,
            ease: 'easeIn',
            onUpdate: (v) => setHaloOpacity(v),
          })
        },
      })

      // After bloom (500ms), float to nav.
      window.setTimeout(() => {
        if (cancelled) return
        const navDot = document.querySelector<HTMLElement>('.nav-avail-dot')
        const anchors = getAnchors()
        if (!navDot || !anchors) {
          // Fallback: just resolve handoff without flight.
          resolveHandoff()
          return
        }
        const navRect = navDot.getBoundingClientRect()
        const targetX = navRect.left + navRect.width / 2
        const targetY = navRect.top + navRect.height / 2
        const startX = anchors.periodX
        const startY = anchors.periodY
        animate(0, 1, {
          duration: 0.7,
          ease: [0.65, 0, 0.35, 1],
          onUpdate: (t) => {
            setFlightPos({
              x: startX + (targetX - startX) * t,
              y: startY + (targetY - startY) * t,
            })
          },
          onComplete: () => {
            // Identity merge: fade the loader cursor out over 100ms while
            // the nav availability dot becomes visible underneath (its CSS
            // pulse takes over). Then unmount.
            animate(1, 0, {
              duration: 0.1,
              onUpdate: (v) => setPostFlightOpacity(v),
              onComplete: () => {
                resolveHandoff()
                setUnmounted(true)
              },
            })
          },
        })
      }, 500)
    })

    return () => {
      cancelled = true
    }
  }, [drawDone, prefersReducedMotion, getAnchors, resolveHandoff])

  if (prefersReducedMotion) return null
  if (unmounted) return null

  const anchors = getAnchors()
  if (!anchors) return null

  const state =
    flightPos !== null
      ? { x: flightPos.x, y: flightPos.y, opacity: postFlightOpacity }
      : computeDuringDraw(progress, anchors)

  return createPortal(
    <div
      className="loading-cursor"
      style={{
        transform: `translate3d(${state.x - 3.5}px, ${state.y - 3.5}px, 0) scale(${dotScale})`,
        opacity: state.opacity,
      }}
      aria-hidden="true"
    >
      <div
        className="loading-cursor-halo"
        style={{
          transform: `scale(${haloScale})`,
          opacity: haloOpacity,
        }}
      />
    </div>,
    document.body
  )
}
```

`App.tsx` currently renders `<LoadingScreen />` as a sibling of `<SmoothScroll>`. After this task it does not — the loader is now an in-place state of `Hero`. We add a small hook that listens for `handoffDone` and toggles `document.body.dataset.loaderState` from `"loading"` to `"done"`, preserving the existing CSS scroll-lock rule (`html:has(body[data-loader-state="loading"])`) so we don't have to change CSS scroll-lock semantics.
```ts
// src/hooks/useScrollLockDuringLoader.ts
import { useEffect } from 'react'
import { useMotion } from '../context/MotionContext'

/** Manages document.body.dataset.loaderState for the entire loader
 *  lifecycle. Sets "loading" on mount; flips to "done" when handoffDone
 *  resolves. The CSS rule in index.css keys off this attribute to lock
 *  scroll on html + body. */
export function useScrollLockDuringLoader(): void {
  const { handoffDone } = useMotion()
  useEffect(() => {
    document.body.dataset.loaderState = 'loading'
    let cancelled = false
    handoffDone
      .then(() => {
        if (cancelled) return
        document.body.dataset.loaderState = 'done'
      })
      .catch(() => {})
    return () => {
      cancelled = true
      // Defensive cleanup: if App unmounts mid-load (HMR), free the lock.
      if (document.body.dataset.loaderState === 'loading') {
        delete document.body.dataset.loaderState
      }
    }
  }, [handoffDone])
}
```

```tsx
      .then(() => { if (!cancelled) setVisible(true) })
      .catch(() => {})
    return () => { cancelled = true }
  }, [handoffDone])

  // ... rest of existing code unchanged ...
```

Then change the `<header>` element to apply opacity:

```tsx
return (
  <header
    className={`nav${scrolled ? ' is-scrolled' : ''}`}
    style={{
      opacity: visible ? 1 : 0,
      transition: 'opacity 200ms ease-out',
      pointerEvents: visible ? 'auto' : 'none',
    }}
  >
    {/* existing nav-inner unchanged */}
  </header>
)
```

The 200ms duration matches the cursor's last 200ms of flight, so the header materializes around the dot exactly as it lands on the nav availability indicator.

```ts
test('body scroll is locked while loader is active', async ({ page }) => {
  await page.goto('/')
  // Loader is up — body should be marked loading and have overflow:hidden.
  const stateDuring = await page.evaluate(() => document.body.dataset.loaderState)
  expect(stateDuring).toBe('loading')

    getComputedStyle(document.body).overflow
```
